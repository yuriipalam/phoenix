## Prerequisites [#building-website-prerequisites]

The website build script has significantly bit-rotted. See [PHOENIX-6867](https://issues.apache.org/jira/browse/PHOENIX-6867).

- Maven `site.xml` must have a proxy set for the `http://conjars.org` repository.
- The `mvn` command must be configured to use Java 1.8. Newer versions do not work.

## Building Phoenix Project Website

1. Make a local copy of source Markdown files and HTML pages.

```shell
git clone git@github.com:apache/phoenix-site.git
```

2. Edit/Add source markdown files in `site/source/src/site/markdown` directory.
3. Edit `phoenix-docs/src/docsrc/help/phoenix.csv` to update reference pages, adding any missing words to `phoenix-docs/src/tools/org/h2/build/doc/dictionary.txt`.
4. Run `build.sh` at the repo root to generate/update HTML pages in the `output` directory.
5. Commit source Markdown files and generated HTML pages, then push your branch.
6. Create a pull request in GitHub for the [mirror of the Phoenix Site repository](https://github.com/apache/phoenix-site), following the GitHub workflow on the [contributing](/docs/contributing) page.

## Local Testing During Development

The site uses protocol-relative URLs for included assets to support `http` as well as `https`. This can cause assets to fail to load when working locally if not using a web server. The root cause is that locally opened files use the `file:` protocol, but some assets live on remote servers thus requiring the `http:` or `https:` protocol.

For best results when testing locally, spin up a simple Python web server after generating the site.

```shell
cd output
python -m SimpleHTTPServer 8000
```

If using Python 3, run:

```shell
python3 -m http.server 8000
```

Now you can access the website at [http://localhost:8000](http://localhost:8000) and your changes are available with a page refresh.
